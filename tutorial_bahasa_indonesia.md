
# Panduan Lengkap Pengembangan & Deployment NOVA

Dokumen ini adalah panduan langkah-demi-langkah untuk membangun, mengonfigurasi, dan meluncurkan aplikasi NOVA.

---

## 1. Pengembangan Frontend (React)

Aplikasi NOVA dikembangkan dengan **React** menggunakan pendekatan *Single Page Application* (SPA).

### Cara Kerja Komponen
- **State Management**: Kita menggunakan `useState` untuk melacak posisi user (View) dan data user yang sedang login.
- **Effect Hook**: `useEffect` digunakan untuk mengambil data dari Supabase segera setelah aplikasi dimuat.
- **Routing Statis**: Navigasi dikelola secara kondisional melalui variabel `view` (misal: `switch(view)`).

### Menjalankan di Lokal
1. Gunakan terminal dan jalankan:
   ```bash
   npx serve .
   ```
2. Aplikasi akan berjalan di `http://localhost:3000`.

## 2. Setup Backend (Supabase)

Supabase adalah alternatif Firebase berbasis PostgreSQL.

### Konfigurasi Database (SQL)
Buka Dashboard Supabase > SQL Editor, dan jalankan perintah ini untuk membuat struktur tabel:

```sql
-- Tabel Profil User
create table profiles (
  id uuid references auth.users on delete cascade primary key,
  full_name text,
  role text default 'user'
);

-- Tabel Event
create table events (
  id bigint generated by default as identity primary key,
  title text not null,
  type text,
  event_date date,
  event_time text,
  price numeric,
  image_url text,
  description text
);

-- Tabel Tiket
create table tickets (
  id text primary key,
  user_id uuid references auth.users,
  event_id bigint references events,
  quantity int,
  total_price numeric,
  status text default 'pending',
  qr_code text,
  created_at timestamp with time zone default now()
);
```

### Ambil API Key
Salin `SUPABASE_URL` dan `SUPABASE_ANON_KEY` dari menu **Project Settings > API**.

## 3. Deployment ke Netlify

Netlify akan menarik kode Anda dari GitHub dan menghostingnya secara otomatis.

1. **Upload ke GitHub**: Masukkan kode Anda ke repositori GitHub.
2. **Koneksi ke Netlify**:
   - Klik **Add New Site** > **Import from existing project**.
   - Pilih repositori GitHub Anda.
3. **Konfigurasi Build**:
   - Jika project menggunakan Vite: Build command: `npm run build`, Publish directory: `dist`.
   - Jika project statis: Biarkan kosong.
4. **Environment Variables**:
   - Masuk ke **Site Settings > Environment Variables**.
   - Masukkan `VITE_SUPABASE_URL` dan `VITE_SUPABASE_ANON_KEY`.

## 4. Menghubungkan Domain Kustom (.com)

Setelah aplikasi online di URL `.netlify.app`, Anda bisa mengubahnya menjadi `.com`.

### Langkah 1: Beli Domain
Beli domain di registrar seperti **Niagahoster**, **Rumahweb** (lokal), atau **Namecheap**, **GoDaddy** (global).

### Langkah 2: Tambahkan Domain di Netlify
1. Di Dashboard Netlify, pilih situs Anda.
2. Pergi ke **Site Configuration > Domain Management**.
3. Klik **Add a custom domain** dan masukkan domain Anda (misal: `nova-engagement.com`).

### Langkah 3: Konfigurasi DNS (PENTING)
Ada dua cara:
- **Cara A (Netlify DNS - Direkomendasikan)**:
  1. Netlify akan memberikan daftar *Nameservers* (misal: `dns1.p01.nsone.net`).
  2. Login ke panel kontrol tempat Anda beli domain.
  3. Ubah **Nameservers** bawaan menjadi milik Netlify.
- **Cara B (A Record)**:
  1. Di panel domain Anda, buat **A Record** yang mengarah ke IP Netlify (`75.2.60.5`).
  2. Buat **CNAME Record** untuk `www` yang mengarah ke URL `.netlify.app` Anda.

### Langkah 4: Aktivasi SSL (HTTPS)
Netlify akan secara otomatis menerbitkan sertifikat **SSL (Let's Encrypt)** secara gratis. Ini biasanya aktif dalam 1-24 jam setelah DNS terhubung.
